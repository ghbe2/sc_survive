# Fable / Sol 共有相談箱

Fable（ゲームデザイン・仕様）と Sol / Codex（技術設計・実装）が、担当外の判断を相談するための公開ログ。
現行仕様の正は `AGENTS.md`。ここは未決事項と回答の受け渡しに使い、確定した恒久仕様は `AGENTS.md` へ反映する。

## 運用ルール

1. 相談IDは `C-YYYYMMDD-NN`。状態は `OPEN / ANSWERED / DECIDED / CLOSED` のいずれか。
2. ゲームの目的・勝敗・操作・キャラ個性など体験を変える判断は、オーナーの `DECIDED` が出るまで実装しない。
3. 実装方法だけの相談は、担当者が回答を採用して `DECIDED` にしてよい。ただし体験や仕様が変わる場合はオーナー判断へ戻す。
4. `OPEN` の相談が作業を遮る場合、推測で進めず停止する。遮らない調査・検証は続けてよい。
5. 回答側は結論だけでなく、理由・トレードオフ・推奨案を書く。往復は原則2回まで。
6. `CLOSED` にした担当者は、必要なら同じコミットで `AGENTS.md` を更新する。
7. このファイルは公開される。キャラ原作本文、非公開URL、秘密情報、成人向けの詳細設定は書かない。

### C-20260717-02 — 到達体験「合奏」システム（野獣/猛獣/百獣）実装依頼

- 状態: DECIDED（オーナー決定済み・実装待ち）
- 起票: Fable
- 回答依頼: Sol（実装担当）
- 関連箇所: `index.html` の tryBurst / updateDevices / executeBurst / endGame / セーブ / タイトル画面
- 期限・優先度: 高（ゲームの「ダブルスイカ」枠。スコアの次の柱）

#### 決定内容（オーナー承認済み仕様）
スイカゲームの「ダブルスイカ」にあたる到達体験。**手動爆発の瞬間に稼働中の置き土産の数**で合奏が成立する。

- 判定: 長押し爆発の発動時、フィールドの置き土産デバイス数（キャラ重複なし）を数え、`1+デバイス数` が合奏人数。
- 成立時、**稼働中の全デバイスは即座に自分の爆発を発動**（timerリセット・leftは通常通り消費）＝画面上で全員の爆発が同時に咲く。
- 格と演出テキスト:
  - 2人 →「"野獣"っぷり！！」
  - 3人 →「"猛獣"の如し！！」
  - 4人 →「"百獣"の王！！」
- 報酬（初期値・実機調整可）:
  - スコア: 人数²×1000（riskMult乗算）
  - 浄化: 三重奏=操作キャラの黒♡1個→白♡に変換 / **四重奏=パーティ全員の黒♡1個ずつ→白♡に変換**（終盤の黒♡蓄積への回復手段。ぱふ先生の浄化と同じ語彙）
  - 二重奏はスコアのみ
- 演出: 人数に応じて画面フラッシュ+大型テキスト（四重奏は特大演出。SEは後日）。
- 記録: `saveData.bestEnsemble`（0〜4）を保存し、リザルトとタイトル画面に最高記録を常設表示（例「最高記録: "猛獣"の如し」）。未達成は「???」表示で煽る。

#### 前提・制約
- 交代=置き土産、爆発CT15秒の現行仕様は変えない（交代時CT半減案は不採用と決定済み）。
- 白♡が多いほどデバイスが長く残る現行仕様が合奏の土台。ここは触らない。

#### 検証観点
- ソロ（デバイス0）で合奏が誤発動しない / 同キャラのデバイスを二重に数えない
- 四重奏の浄化が離脱メンバーに作用しない（生存者のみ）
- bestEnsembleのセーブとタイトル/リザルト表示
