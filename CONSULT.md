# Fable / Sol 共有相談箱

Fable（ゲームデザイン・仕様）と Sol / Codex（技術設計・実装）が、担当外の判断を相談するための公開ログ。
現行仕様の正は `AGENTS.md`。ここは未決事項と回答の受け渡しに使い、確定した恒久仕様は `AGENTS.md` へ反映する。

## 運用ルール

1. 相談IDは `C-YYYYMMDD-NN`。状態は `OPEN / ANSWERED / DECIDED / CLOSED` のいずれか。
2. ゲームの目的・勝敗・操作・キャラ個性など体験を変える判断は、オーナーの `DECIDED` が出るまで実装しない。
3. 実装方法だけの相談は、担当者が回答を採用して `DECIDED` にしてよい。ただし体験や仕様が変わる場合はオーナー判断へ戻す。
4. `OPEN` の相談が作業を遮る場合、推測で進めず停止する。遮らない調査・検証は続けてよい。
5. 回答側は結論だけでなく、理由・トレードオフ・推奨案を書く。往復は原則2回まで。
6. `CLOSED` にした担当者は、必要なら同じコミットで `AGENTS.md` を更新する。
7. このファイルは公開される。キャラ原作本文、非公開URL、秘密情報、成人向けの詳細設定は書かない。

```md

- 状態: OPEN
- 起票: Fable / Sol / オーナー
- 回答依頼: Fable / Sol / オーナー
- 関連箇所: `index.html` の機能名など
- 期限・優先度: 任意

#### 質問
何を決める必要があるか。

#### 前提・制約
- 変えてはいけない仕様
- 技術上または体験上の制約

#### 選択肢
1. 案A — 長所 / 短所
2. 案B — 長所 / 短所

#### 起票者の推奨
推奨案と理由。判断不能なら「なし」。

#### 回答
回答者が追記。結論、理由、注意点を書く。

#### オーナー決定
体験・仕様変更の場合にオーナーが追記。

#### 実装結果
- commit: 未実装
- 検証: 未実施
```
