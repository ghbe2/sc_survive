# SugarSurvive (十二支サバイバー) — エージェント向けガイド

Claude Code (Fable) と Codex が共同で開発するリポジトリ。このファイルが現行仕様の正。
コードを触る前に必ず読み、ここに反する変更はオーナーの明示指示があるときだけ行うこと。

## プロジェクト概要
- スマホ縦持ち向けヴァンサバ風 × スコアアタック型アクション。**単一 `index.html`・依存ライブラリなし・ビルド工程なし**。この制約は維持する。
- デプロイ: `master` に push → GitHub Pages `https://ghbe2.github.io/sc_survive/` が自動更新（1〜2分）。
- UIテキストは日本語。成人向け（センシ）表現を含む。将来 WORDS 辞書方式で全年齢版と切替予定（未実装）。
- 動作確認はブラウザで `index.html` を開くだけ（静的サーバー任意）。

## 遊びのコア（現行の正）
- 操作: ドラッグ=移動 / 単タップ=交代 / 長押し=爆発（白♡1消費）/ 停止=チャージ / 被弾=ゲージ上昇。
- 快感ゲージ100到達で♡1獲得: **自力チャージ→白♡**（爆発の弾薬）/ **敵起因→黒♡**（消せない。例外=ぱふ先生の浄化）。
- **白+黒の合算が♡上限に達すると離脱（蹂躙）**。全員離脱で敗北。総♡数ぶん移動が鈍る（-15%/個、下限25%）。
- 爆発: **クールタイム15秒**（`BURST_COOLDOWN` 900F、冷却スプレーで最短6秒）。共通効果=小ノックバック+短無敵（緊急脱出）。個性は `executeBurst()` の12分岐。
- **交代=置き土産**: 白♡持ちで交代すると、その場に残り白♡を使い切るまでCTごとに自分の爆発を反復。バフ系爆発は操作中キャラに乗る。置き土産中キャラへ切り替えるとその置き土産は強制終了。
- **合奏**: 手動爆発時に稼働中の置き土産があれば全員が即時発火。2/3/4人で野獣/猛獣/百獣となり、人数²×1000点。3人は操作キャラ、4人は生存者全員の黒♡1個を白♡へ浄化。最高記録を保存する。
- 人数別威力倍率: **1人×4 / 2人×3 / 3人×2 / 4人×1**（`CONFIG.PARTY_ATK`）。
- 経済/成長: ❤いいねが経験値（敵が必ず1個ドロップ）。4/8/16/32…でLEVEL UP後、生存メンバー（ヒロイベオ除外）2〜3人から強化対象を選ぶ。20%で回復便。選択後は📦で届き、キャラLv2〜10は固定攻撃表、Lv11以降は小強化。
- 控えの生存メンバーは快感ゲージが+0.03/Fで99まで自然充電。操作キャラの停止チャージは+0.07/F。
- 合奏成立ごとに絆を人数-1獲得。1スタックにつき移動+3%・爆発CT-4%（下限5秒）・チャージ+3%。
- 敵HPは時間で増えない。1分🐆、2分🦏、3分🦍が追加され、数と種類で圧力を上げる。50体撃破ごとに固定HP500のエリートが出現し、撃破で🎁（ランダムな生存メンバーLv+1）+❤5個。
- SEは外部音源なしのWebAudio合成。メニューのON/OFFをセーブし、デフォルトON。

## キャラクター（12人）
- **データ駆動**: `CHAR_DEFS` 配列がすべて。設計の正はオーナー管理のスプレッドシート（このリポジトリには置かない。変更はオーナー経由で受け取る）。
- 単位: 射程1 ≈ 110px（`RANGE_UNIT`）/ 威力はマユゥ10基準 / 頻度=発/秒。
- 攻撃 `atk.type`: `line`（直線）/ `fanRandom`（前方扇ランダム）/ `sweep`（扇形の面）/ `aura`（自分中心の円）/ `boomerang` / `zone`（足元設置）。`bounce`=跳弾、`pierce`=貫通。
- パッシブ `pas` / 攻撃ヒット時 `onHit`（slow=ラコ, poison=アウネ）/ 爆発は `executeBurst()` の case 0〜11。
- ヒロイベオ(id6)は「攻撃しない」設定だが人数倍率と噛ませるため威力2の豆鉄砲を暫定付与中。

## コード構造（index.html 約1300行）
`<style>` → `<body>`（シーンDOM: shop=タイトル / setup=編成 / game-ui / level-scene / menu-scene / result）→ `<script>`:
CONFIG（全バランス定数）→ BUFF_META → CHAR_DEFS / CHAR_LVUPS → ENEMY_TYPES / MISSION_DEFS → セーブ・SE → シーン管理 → 編成UI → ラン状態 → ジェスチャー入力 → 実効値 / 交代・置き土産 / 爆発 → ゲージ・♡ → メニュー / レベルアップ / 宅配 → メインループ（**固定タイムステップ60fps**、accumulator方式）→ update系 → draw → HUD → リザルト

## コーディング規約
- バランス数値は必ず `CONFIG` に置く（マジックナンバー禁止）。
- 配列要素の削除は `dead` フラグ + update末尾の `filter`（forEach内spliceは禁止）。
- シーン切替は `showScene()`、`location.reload()` によるシーン遷移は禁止。
- 固定タイムステップを壊さない（ロジックはフレーム数基準60fps、描画はrAF）。
- 単一HTML・依存なし・ビルドなしを崩さない。フレームワーク導入禁止。

## 作業ルール（マルチエージェント運用）
- **最初に `git status --short --branch` で他エージェントやオーナーの未コミット変更を確認する**。見覚えのない変更は消さず、重なる場合は作業を止めて報告する。
- 作業ツリーがクリーンな場合だけ、実装前に `git pull --ff-only` で最新化する。自動マージや rebase は行わない。
- Claude Code と Codex が同じ `index.html` を同時編集しないよう、1タスクずつ担当を分ける。小さくコミットし、検証後に push する。
- force-push 禁止。コンフリクトしたら手を止めてオーナーに報告。
- コミットメッセージは日本語1行。push前に `git diff --check` と、変更に応じたブラウザ確認を行う。
- このリポジトリは**公開**。センシティブな設計メモ・キャラ原作テキストはコミットしない。
